TILT Technology Inventory
===========================

Complete list of technologies, frameworks, libraries, and important choices used in this project. Grouped by category for easy reference.

=========================================
1. FRONTEND STACK
=========================================

React 18.2.0
- Main frontend framework
- Component based architecture
- Functional components with hooks
- No class components, hooks only

React DOM 18.2.0
- React rendering for web browsers
- Standard React web dependency

Vite 5.0.8
- Build tool and development server
- Fast HMR hot module replacement
- ES modules by default
- Replaces Create React App or Webpack
- Dev server on port 3000

JavaScript ES6+
- Modern JavaScript features
- ES modules import export
- Arrow functions, destructuring, async await
- No TypeScript, plain JavaScript

JSX
- React component syntax
- HTML like syntax in JavaScript
- Transpiled by Vite

=========================================
2. BACKEND STACK
=========================================

Python 3.12+
- Backend programming language
- Type hints used but not enforced
- Standard library plus third party packages

FastAPI 0.104.1
- Modern Python web framework
- REST API endpoints
- Automatic OpenAPI documentation
- Pydantic models for validation
- Async support though not heavily used

Uvicorn 0.24.0
- ASGI server for FastAPI
- Runs on port 8000
- Standard variant with all features
- Handles HTTP requests and WebSocket support

Pydantic 2.5.0
- Data validation library
- Request response models
- Type checking and validation
- Used for API request response schemas

=========================================
3. DATABASE
=========================================

SQLite3
- Embedded database engine
- Single file database tilt_data.db
- No separate database server required
- Python sqlite3 standard library
- SQLCipher planned but not implemented

Database Schema
- vendors table for vendor data
- questions table for standard assessment questions
- cloud_criteria table for cloud scorecard criteria
- settings table for application settings
- Migrations handled in database.py

=========================================
4. SECURITY & AUTHENTICATION
=========================================

Argon2
- Password hashing algorithm
- Via passlib library with argon2 backend
- Resists GPU cracking attacks
- Industry standard for password hashing
- Hash stored in .tilt_key file

Master Password System
- Single password protects entire database
- Required for all API operations
- No password recovery by design
- Password verification on every request
- Stored as Argon2 hash, not plaintext

CORS Middleware
- Cross origin resource sharing
- Allows localhost:3000 to access localhost:8000
- Development only configuration
- Should be restricted in production

=========================================
5. STYLING & UI
=========================================

Tailwind CSS 3.4.0
- Utility first CSS framework
- No custom CSS files needed
- Responsive design utilities
- Custom configuration in tailwind.config.js

PostCSS 8.4.32
- CSS processing tool
- Used by Tailwind for transformations
- Autoprefixer integration

Autoprefixer 10.4.16
- Automatic vendor prefixes
- Browser compatibility
- Integrated with PostCSS

Custom Fonts
- Poppins from Google Fonts
- Poppins Black for main headings
- Poppins Thin for section titles and labels
- Poppins Regular for default body text
- Font files in Poppins Fonts directory
- Applied via @font-face in index.css

Lucide React 0.344.0
- Icon library
- React components for icons
- Used for UI icons like Save, X, AlertTriangle

6. OSINT & SECURITY CHECKS
=========================================

Requests 2.31.0
- HTTP library for Python
- Fetches ransomware list from GitHub
- Handles JSON responses
- Timeout and error handling

dnspython 2.4.2
- DNS query library
- DMARC record lookups
- TXT record parsing
- DNS resolver functionality

SSL/TLS Validation
- Python standard library ssl module
- Python standard library socket module
- Certificate expiration checking
- TLS handshake validation
- Port 443 connections

Ransomwatch GitHub
- External data source
- Free open source ransomware leak database
- JSON API endpoint
- No authentication required

DNS Protocol
- Standard DNS queries
- TXT record lookups for DMARC
- Public DNS resolution
- No special DNS servers required
